DebuggingÂ¶

yall defines some debugging function, useful to avoid messing with the susbystems tree or to have more informations about an error.

First, debug mode should be enabled :

_YALL_PUBLIC yall_error yall_enable_debug(const char * subsystem)

Enable the debug mode on the library. This function must be called AFTER yall_init(). Debug message will be wrote using the given subsystem.

Return
Error code whether it succeed.
Parameters
  • subsystem: Name of the subsystem to us as debug subsystem. If the subsystem does not exists, no debug message will be generated. Also, this subsystem should not be modified during debug mode.

The given subsystem must exists before using it as debug subsystem. Once debug mode activated, yall will write log message (using this subsystem) during its use.

To check debug mode status :

_YALL_PUBLIC bool yall_is_debug(void)

Return the status of the debug mode.

Return
Boolean whether the debug mode is initialized.

If you need to disable debug mode (it does not delete the debug subsystem) :

_YALL_PUBLIC yall_error yall_disable_debug(void)

Disable debug mode. The debug subsystem is not removed nor modified.

Return
Error code whether it succeed.

Finally, it is possible to display the current subsystems tree with :

_YALL_PUBLIC void yall_show_subsystems_tree(void)

Display the subsystems tree. Supports up to 32 nested subsystems. This function is not thread safe.

Remark
This function can only be called when debug mode is activated.
