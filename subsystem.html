Subsystem¶

Creation¶

Subsystems are the major feature of the library as they store different sets of logging parameters. Creating a subsystem is defining a set of parameter about how to handle log messages affected to this specific subsystem.

Subsystem creation is done with :

_YALL_PUBLIC yall_error yall_set_subsystem(const char * name, const char * parent, enum yall_log_level log_level, enum yall_output_type output_type, const char * output_file)

This is the main subsystems function. This function handle subsystems creation and update. If name is the name of an existing subsystem, it is updated : removed from the subsystems tree, updated and replaced in its correct position with correct inheritance. If name is not the name of an existing subsystem, it is created with the proper parameters and inserted in the subsystems tree at the proper place.

Return
Error code depending of the success or failure of the function.
Parameters
  • name: Name of the subsystem to add or update. Can’t be NULL.
  • parent: Name of the parent to inherit from. If NULL, this subsystem will be at top-level of the tree.
  • log_level: Minimum log level to use for this subsystem. All the log messages with a log level below this one will be discarded.
  • output_type: Default output type for this subsystem. See yall_output_type.
  • output_file: File to use of the subsystem’s default output type is yall_file_output.

If the subsystem name already exists, it will be updated with the given parameters.

Each subsystem can have, a parent from which it can inherits some parameters.

log_level can have one of the following values :

enum yall_log_level

This enumerator stores the different log level used by the library. They are extracted from the Syslog protocol (RFC 5424). The last one : yall_inherited_level is used to inform that a subsystem inherit its log level from its parent.

Values:

yall_debug
yall_info
yall_notice
yall_warning
yall_err
yall_crit
yall_alert
yall_emerg
yall_inherited_level

output_type can have one of the following values :

enum yall_output_type¶

This enum stores all the different output types for the subsystems. These values has to be power of 2, as they can be combined to write on multiple outputs.

Values:

yall_inherited_output = 0¶

Subsystem inherit its output type from its parent.

yall_console_output = 1 << 0¶

Subsystem log message will be wrote on console.

yall_file_output = 1 << 1¶

Subsystem log message will be wrote in file.

yall_syslog_output = 1 << 2¶

Subsystem log message will be redirected to syslog.

If output_type is yall_file_output, then output_file can be set to the output file name, or to NULL to use the parent’s output file.

When using yall_syslog_output, the formatted output can be broken if using line feed. To avoid this, update syslog configuration to disable control characters escaping (on rsyslog, parameter is parser.escapeControlCharactersOnReceive).

Destruction¶

Subsystems are destroyed during library deinitialization. Currently, their is no way to specifically destroy a subsystem during execution.
