30 /*
31  * Theses macros are used to generate log messages. There is two types of
32  * logs :
33  * * YALL_<log_level>(<subsystem>, <format>, ...) : which will print
34  * the log message according the given subsystem's parameters.
35  * If the subsystem does not exists, default parameters are
36  * used. <format> is a printf() like format, with the parameters.
37  * * YALL_<log_level>_IF(<subsystem>, <condition>, <format>, ...) : same
38  * function as the previous, but the log will be processed only
39  * if the condition is true.
40  * * YALL_CALL_<log_level>(<subsystem>, <function>, <args>) : with these
41  * macros, the format of the message will be handled by the given
42  * function. <function> will be called with <args> as parameters.
43  * * YALL_CALL_<log_level>_IF(<subsystem>, <expr>, <function>, <args>) :
44  * same function as the previous, except that the message will if
45  * created and displayed only if <expr> evaluates to true.
46  *
47  * The _YALL_LOG(<...>) wrapper is used to handle C++ stream operators. This
48  * way, on a C++ compiler, we can call the logging function like :
49  * YALL_DEBUG(sub, "This is " << "my log " << "msg");
50  */
