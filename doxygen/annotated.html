C lightweight logging library
Data Structures
Here are the data structures with brief descriptions:
 Ccqueue_tThis structure represents a concurrent queue
 Cheader_contentStructure representing all the possible data that can be write inside a log message header. The header generation function can then get the header values from it
 CmessageThis is a message. It contains a log line and its output parameters (gathered from the proper subsystem). This structure is used to store the different log messages inside a lockless queue, waiting to be wrote on the medium. It should be avoided to create a message manually, use message_new() instead
 CqnodeThis structure is used to represent a queue node
 Cyall_call_dataStructure storing calling system data. An instance of this structure is passed as a parameter to the formatter function during calls to YALL_CALL_xxx macros. All these values shouldn't be modified by the formatter
 Cyall_call_data_lineRepresent a line during a call to a macro YALL_CALL_xxx
 Cyall_configThis structure store the current configuration of the library. Each configuration parameter we had should be added to this structure
 Cyall_console_output_configThis structure is used to store the set of console output type parameters. Currently, no parameters are available for console output type
 Cyall_file_output_configThis structure is used to store the set of file output type parameters
 Cyall_subsystemThis structure contains all the parameters and configuration for a given subsystem
 Cyall_subsystem_paramsThis structure is used as a placeholder for parameter during the processing of a log message. It is not possible to directly use the struct yall_subsystem parameters as it could contains inherited values
