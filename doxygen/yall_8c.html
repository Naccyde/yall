Another logging function which call the given function parameter to change message format. It will get the subsystem's parameters and use them to add header on the logging message. Then it will call the given function with a buffer to fill.

Parameters
subsystemName of the calling function, defaultly used in the log message's header. Can't be NULL.
log_levelLog level to use for this log message.
function_nameName of the calling function, defaultly used in the log message's header. Can't be NULL.
filenameName of the file calling the log function.
lineLine on which this function has been called.
formatterFunction which will handle the logging of the message. The given d parameter of type yall_call_data will contain of the necessary data for yall to process the log message. Inside this formatting function yall_call_set_header() and yall_call_add_line() can be called.
argsvoid * pointer to custom arguments to passe to the formatter function.
Returns
Error code depending of the success or failure of the function.

Major logging function, used to forge the message and write it on the medium. Given the parameters, it will decide if the message should be logged or not. This function should not be used as it but through all the macro (YALL_DEBUG(), ...).

Parameters
subsystemName of the subsystem as a nul-terminated string. Can't be NULL.
log_levelLog level to use for this log message.
functionName of the calling function, defaultly used in the log message's header. Can't be NULL.
filenameName of the file calling the log function.
lineLine on which this function has been called.
formatFormat of the log message, in a printf() way. Can't be NULL.
...Variadic parameters to use with the described defined format. All the printf() like functions modifiers can be used.
Returns
Error code depending of the success or failure of the function.

◆ yall_set_subsystem()

yall_error yall_set_subsystem ( const char *  name,
const char *  parent,
enum yall_log_level  log_level,
enum yall_output_type  output_type,
const char *  output_file 
)

This is the main subsystems function. This function handle subsystems creation and update. If name is the name of an existing subsystem, it is updated : removed from the subsystems tree, updated and replaced in its correct position with correct inheritance. If name is not the name of an existing subsystem, it is created with the proper parameters and inserted in the subsystems tree at the proper place.

Parameters
nameName of the subsystem to add or update. Can't be NULL.
parentName of the parent to inherit from. If NULL, this subsystem will be at top-level of the tree.
log_levelMinimum log level to use for this subsystem. All the log messages with a log level below this one will be discarded.
output_typeDefault output type for this subsystem. See yall_output_type.
output_fileFile to use of the subsystem's default output type is yall_file_output.
Returns
Error code depending of the success or failure of the function.
